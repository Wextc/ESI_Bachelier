## TD4 et TD5 – DDL-DML

<b>Création de table:</b>

```
CREATE TABLE nom_table (
    nom_col1 type contrainte(s),
    nom_col2 type contrainte(s),
    ...
);

```

<b>Sppréssion de table:</b>

```
DROP TABLE nom_table;

```

<b>Modification de table:</b>

```
ALTER TABLE nom_table ADD nom_col type contrainte;

```

<b>Insertion:</b>

```
INSERT INTO nom_table (nom_col1, nom_col2,... , nom_colN)
VALUES (val1, val2, ... valN);

Ou

INSERT INTO nom_table2 (nom\_col1, nom_col2,... ,nom_colN)
SELECT nom_col1, nom_col2, ..., nom_colN
FROM nom_table1
WHERE condition;

```

<b>Supprésion de lignes:</b>

```
DELETE FROM nom_table
WHERE condition_sur_les_lignes;

```

<b>Modification de ligne:</b>

```
UPDATE nom_table
SET nom_col1 = val1, nom_col2 = val2, ..., nom_colN = valN
WHERE condition_sur_les_lignes;

```

Exercice 1:

```
Exécuter la requête suivante :
CREATE TABLE Test (
tId int NOT NULL,
tLib varchar(50) NOT NULL,
tNb1 decimal(5,2) DEFAULT 12 NOT NULL,
tNb2 decimal(8,2) NOT NULL
);

```

Elle va créer une table Test avec 4 attributs avec:

    Le type varchar(n) définit une chaîne de caractères de longueur variable mais de maximum n caractères.

    Le type decimal(p,s) définit un réel de maximum p digits dont s derrière la virgule.

    Tous les attributs sont obligatoires (NOT NULL).

    Si on ne donne pas de valeur à tNB1 il prendra par défaut la valeur 12.

```
CREATE TABLE Test (
    tId int NOT NULL,
    tLib varchar(50) NOT NULL,
    tNb1 decimal(5,2) DEFAULT 12 NOT NULL,
    tNb2 decimal(8,2) NOT NULL
);
```

---

Exécuter la commande suivante : desc Test qui permet d’afficher la description de la table Test. Vérifier que vous avez bien créé convenablement la table.

```
desc Test

```
